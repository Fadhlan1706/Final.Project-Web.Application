## Final.Project-Web.Application<br>
Tugas Kelompok - Project dikerjakan secara kelompok. Membuat aplikasi web dinamis (Sistem informasi web sederhana). Menerapkan HTML, CSS, JS, PHP & MySQL yang sudah dipelajari. Presentasi (sesuai pengaturan kelas masing-masing)<br>
**Batas submit project: Minggu, 07 Juni 2026 23.59 Wita**

Tugas kelompok dikerjakan dalam sebuah tim yang terdiri dari (3-4 orang). Setiap kelompok akan membagi tugas untuk masing-masing anggota dalam mengerjakan project ini. 

**Tujuan**<br>
Membuat aplikasi web dinamis yang multi-user dan multi-konten atau sistem informasi sederhana berbasis web. Tema project bebas sesuai kreativitas kelompok masing-masing.

**Requirement Specification**<br>
1. Menerapkan HTML, CSS, JS, PHP, MySQL dalam project ini.
2. Memiliki setidaknya (minimum): Halaman static & halaman dinamis
   - Halaman dinamis yang menampilkan data, yang diambil dari database MySQL.
   - Halaman statis tentang informasi anggota-anggota kelompok.
   - Mekanisme login untuk admin atau member untuk mengakses halaman khusus.
   - Halaman untuk manajemen konten (Tambah, Edit, Hapus).
   - Halaman lain yang berisi konten bebas lainya yang terkait dengan tema project, sesuai kreatifitas kelompok masing-masing.
4. Tidak menggunakan library atau framework apapun, contoh JQuery, Bootstrap, CodeIgniter, dll.
5. Project WAJIB di-deploy pada suatu server hosting tertentu dan dapat diakses secara online (live web)

**Isi Dokumen Final Project**<br>
1. Screenshoot dan penjelasan tentang
   - Halaman dinamis yang menampilkan data, yang diambil dari database MySQL.
   - Halaman statis tentang informasi anggota-anggota kelompok.
   - Mekanisme login untuk admin atau member untuk mengakses halaman khusus.
   - Halaman untuk manajemen konten (Tambah, Edit, Hapus).
   - Halaman lain yang berisi konten bebas lainya yang terkait dengan tema project, sesuai kreatifitas kelompok masing-masing.
2. Deskripsi Aplikasi – jelaskan tentang latar belakang, tujuan utama aplikasi, manfaat dan fitur aplikasi secara umum.
3. Users (aktor) – daftar pengguna dan deskripsi singkat tentang masingmasing User dan perannya dalam menggunakan aplikasi. Disajikan dengan Diagram Use Case.
4. Screenshoot dan penjelasan pendukung lainnya.

**Teknis Pengumpulan Tugas**<br>
1. Project di-deploy pada suatu server hosting tertentu yang dapat diakses secara online.
2. Laporan Project di-submit di E-Learning sebelum batas waktu.
3. Setiap kelompok akan mempresentasikan project melalui mekanisme dan jadwal sesuai kelas masing-masing. (Zoom/Video Rekaman/Presentasi di Kelas).
